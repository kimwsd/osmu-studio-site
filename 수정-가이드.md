# OSMÜ STÜDIO 홈페이지 수정 가이드

이제 사이트는 **여러 개의 페이지(파일)** 로 나뉘어 있습니다.
디자인과 동작은 공통 파일 두 개(`osmu.css`, `osmu.js`)가 모든 페이지에서 함께 쓰입니다.
즉, 색상·폰트·메뉴 동작을 한 번만 고치면 전체 페이지에 동시에 반영됩니다.

## 파일 구성

| 파일 | 역할 |
|------|------|
| `osmu-studio.html` | **메인(홈)** 페이지 |
| `work.html` | 프로젝트 전체 목록 |
| `studio.html` | 스튜디오 소개 |
| `services.html` | 서비스 소개 |
| `contact.html` | 문의 페이지 (메일 폼) |
| `project-*.html` | 프로젝트별 상세 페이지 (6개) |
| `admin.html` | 관리자 페이지 |
| `osmu.css` | **공통 디자인(색상·레이아웃)** |
| `osmu.js` | **공통 동작(메뉴·애니메이션·프로젝트 데이터)** |
| `logo.png` | 로고 이미지 (모든 페이지 공용) |

> ⚠️ 페이지들은 같은 폴더 안에 있어야 서로 연결됩니다. 옮길 때는 **폴더 통째로** 옮기세요.

## 자주 수정할 위치

### 1. 색상 테마 — `osmu.css` 맨 위
`:root { ... }` 부분
- `--white` : 배경색
- `--ink` : 글자/포인트 색 (현재 #0a0a0a)
- `--grey`, `--line` : 보조 회색, 구분선

여기만 고치면 모든 페이지 색이 한 번에 바뀝니다.

### 2. 프로젝트 추가/수정 — `osmu.js` 맨 위
`const PROJECTS = [ ... ]` 부분
```js
{slug:'cafe-mono', name:'CAFE MONO', cat:'Space + Branding', type:'카페', year:2026, month:3, loc:'아산 Asan'},
```
- `slug` : 상세 페이지 파일 이름과 연결됩니다. (`slug:'cafe-mono'` → `project-cafe-mono.html`)
- `cat` : 작업 분류 (Space + Branding / Space Design / Brand Identity / Packaging / Marketing)
- `type` : **업종 / 공간 유형** (카페 / 주거 / 상업 / 음식점 / 술집 / 기타)
- `year` / `month` : 연도와 월 (월은 비워도 됨 → 연도만 표시)
- 이 줄을 추가하면 홈과 `work.html`의 목록에 자동으로 나타납니다.
- 업종 보기 목록은 `osmu.js`의 `PROJECT_TYPES` 배열에서 추가/수정합니다.

**상세 페이지까지 만들려면** 기존 `project-cafe-mono.html` 같은 파일을 복사해
파일명을 새 `slug`에 맞게 바꾸고(`project-새이름.html`), 안의 텍스트(제목·설명)를 고치면 됩니다.
(목록에만 추가하고 상세 페이지를 안 만들면, 클릭 시 "페이지 없음"이 뜹니다.)

#### 사진 업로드 (관리자 페이지)
`admin.html`에서 프로젝트마다 **사진을 여러 장 업로드**할 수 있습니다.
- '＋ 사진 선택 / 추가'로 여러 장을 한 번에 올릴 수 있고, **첫 번째 사진이 대표 이미지(COVER)** 가 됩니다.
- 썸네일을 클릭하면 그 사진을 대표로 지정하고, 우측 상단 × 로 삭제합니다.
- 업로드한 사진은 WORK 목록(마우스 오버 미리보기)과 프로젝트 상세 페이지의 메인 이미지·갤러리에 자동 반영됩니다.
- 업로드 시 사진은 자동으로 적당한 크기로 축소되어 저장됩니다.

> ⚠️ **중요 — 저장 범위:** 관리자에서 추가/수정한 내용과 업로드한 사진은 **작업한 그 브라우저 안(localStorage)** 에만 저장됩니다.
> 같은 브라우저에서는 새로고침해도 유지되지만, **다른 사람·다른 기기에는 보이지 않습니다.**
> 모든 방문자에게 보이도록 실제로 게시하려면 백엔드/CMS(예: Netlify CMS, Supabase, Firebase) 연동이 필요합니다.
> ('초기 데이터로 되돌리기' 버튼으로 업로드한 내용을 모두 지우고 처음 상태로 되돌릴 수 있습니다.)

### 3. 텍스트 내용
- 홈 히어로 태그라인: `osmu-studio.html`의 `<p class="hero-tag">` 안 `<span>` 들
- 스튜디오 소개 문구: `studio.html`
- 서비스 설명: `services.html`의 각 `.svc-body` 안 `<p>`, 그리고 `.process` 단계들
- 연락처/주소/이메일: `contact.html` (그리고 홈의 Contact 섹션)
- 프로젝트 상세 내용: 각 `project-*.html`
- 푸터 문구: 각 파일 맨 아래 `<footer>` 안

### 4. 관리자 계정 — `admin.html`
파일 안에서 `osmu2026` 을 검색하면 로그인 검증 부분이 나옵니다.
```js
if(id==='admin' && pw==='osmu2026'){
```
여기서 아이디/비밀번호를 바꾸세요.
(주의: 클라이언트 코드라 보기엔 노출됩니다. 실서비스에서는 서버 인증이 필요합니다.)

### 5. 애니메이션 타이밍 — `osmu.css`
홈 히어로 인트로 시퀀스 조절값:
- 커튼 올라가는 시점: `.loader` 의 `animation: curtain 1s ... 1.5s`
- 로고 와이프: `.hero-logo img` 의 `1.2s ... 1.9s`
- 태그라인 시차: `.hero-tag>span:nth-child(n)` 의 `animation-delay`
- 전부 빠르게 하려면 각 delay 값을 줄이면 됩니다.

### 6. 로고 교체
이제 로고는 `logo.png` 파일 하나로 관리됩니다. (476×80, 투명 배경)
이 파일을 같은 이름의 새 이미지로 덮어쓰면 모든 페이지의 로고가 바뀝니다.
(어두운 배경의 푸터에서는 CSS가 자동으로 흰색으로 반전해서 보여줍니다.)

### 7. 문의 폼 (`contact.html`)
현재는 '문의 보내기'를 누르면 방문자의 메일 앱이 열리는 방식입니다(서버 불필요).
실제로 폼 내용을 받아 저장하려면 Formspree, Google Forms 같은 외부 서비스나 서버 연동이 필요합니다.
`contact.html` 맨 아래 `<script>` 의 주석에 위치를 표시해 두었습니다.

## 배포
폴더 전체를 호스팅(Netlify, Vercel, 카페24 등)에 올리면 그대로 동작합니다.
시작 페이지를 홈으로 만들려면 `osmu-studio.html` 을 `index.html` 로 복사(또는 이름 변경)해서 함께 올리세요.
(이름을 바꿀 경우, 다른 페이지들의 `href="osmu-studio.html"` 링크도 `index.html` 로 함께 바꿔야 합니다.)

## (예정) CMS 연동 — 추후 작업
> 현재 어드민의 프로젝트·사진은 작업한 브라우저(localStorage)에만 저장됩니다.
> **모든 방문자에게 게시**하려면 CMS/백엔드 연동이 필요하며, 아래 계획대로 진행합니다.

**상태:** 추후 진행 예정 (지금은 localStorage 데모 버전으로 동작 중)

**연동 시 손댈 곳 — 단 한 군데:**
- `osmu.js` 의 `loadProjects()` / `saveProjects()` 두 함수.
  지금은 localStorage를 읽고 쓰는데, 이 부분만 CMS API(fetch) 호출로 교체하면
  나머지(WORK 목록·상세 페이지·어드민)는 코드 변경 없이 그대로 동작합니다.
- 데이터 구조는 유지: `{slug, name, cat, year, loc, images:[...]}`
- 사진은 base64 대신 CMS가 돌려주는 **이미지 URL**을 `images` 배열에 넣으면 됩니다.

**후보 (정적 사이트에 적합한 순):**
1. **Netlify CMS / Decap CMS** — 깃 기반, 어드민에서 올리면 자동 커밋·배포 (별도 서버 불필요, 가장 가벼움)
2. **Supabase / Firebase** — DB + 이미지 스토리지, 지금 어드민 UI 거의 그대로 활용 가능
3. **Sanity / Contentful** — 본격 헤드리스 CMS

**진행 전 결정할 것:** 최종 호스팅 위치(Netlify/Vercel/카페24 등) → 거기에 맞춰 CMS를 고릅니다.
